Cómo ejecutar las pruebas Newman
desde tu computador

Pre-requisitos, instalación y ejecución paso a paso para Entel y ClaroVTR en paralelo
Windows 10 / 11 PowerShell Newman CLI APIM PRE-PROD

1Pre-requisitos — qué debes tener instalado

Antes de ejecutar cualquier prueba, tu computador necesita dos programas instalados.

1.1 Node.js

Node.js es el motor que hace correr Newman. Si no lo tienes instalado:

  1. Ir al sitio oficial de Node.js

    Ingresar a nodejs.org y descargar la versión LTS (la que dice "Recomendado para la mayoría").

  2. Ejecutar el instalador

    Abrir el archivo descargado (.msi) y seguir los pasos. Dejar todas las opciones por defecto y hacer clic en Next hasta finalizar.

  3. Verificar que quedó instalado

    Abrir PowerShell y ejecutar:

    PowerShell — verificar Node.js
    node --version
    ✓ Debe mostrar algo como v20.x.x. Si aparece un número de versión, está OK.

1.2 Newman y el reporter HTML

Con Node.js instalado, abrir PowerShell y ejecutar este único comando que instala ambas herramientas a la vez:

PowerShell — instalar Newman
npm install -g newman newman-reporter-htmlextra
ℹ El proceso tarda aproximadamente 1–2 minutos. Es normal que aparezcan muchas líneas de texto mientras instala.

1.3 Verificar que todo quedó listo

PowerShell — verificar instalación
newman --version
✓ Si aparece un número de versión (ej: 6.x.x), ya está todo listo. Solo se hace esta instalación una vez.

2Archivos necesarios en tu computador

Debes tener una carpeta llamada Kommand con exactamente esta estructura:

Kommand/
│
├── run_newman.ps1  ← el script que ejecuta las pruebas
│
└── collection Blueplanet/
    ├── Komands — APIM PRE VNOs 02-03 Claro-Entel (Auto-Token).postman_collection.json
    │       ← la colección con los 5 requests
    ├── VnoB1_vnoid03 PRE.postman_environment.json
    │       ← variables de ambiente Entel
    └── VnoB1_vnoid02 PRE ClaroVTR.postman_environment.json
            ← variables de ambiente ClaroVTR
⚠
Estos dos archivos no se incluyen al descargar el repositorio porque contienen contraseñas de acceso al sistema.

Debes pedirlos al responsable del proyecto, quien te los enviará por un canal seguro interno (Teams, carpeta compartida, etc.). Una vez que los tengas, cópialos manualmente dentro de la carpeta collection Blueplanet\.

3Cómo ejecutar las pruebas

3.1 Abrir PowerShell en la carpeta correcta

La forma más fácil es desde el Explorador de Windows:

  1. Abrir la carpeta Kommand en el Explorador de Windows
  2. Clic derecho en un espacio vacío dentro de la carpeta

    Seleccionar "Abrir en Terminal" o "Abrir ventana de PowerShell aquí".

  3. Se abre PowerShell ya posicionado en la carpeta correcta
ℹ Si prefieres abrir PowerShell manualmente, ejecutar: cd "C:\ruta\a\Kommand" para ir a la carpeta antes de ejecutar el script.

3.2 Ejecutar con los valores por defecto

Si quieres probar con los valores que ya tiene el script configurados:

PowerShell
.\run_newman.ps1

3.3 Ejecutar con tus propios datos de prueba

Cada vez que se prueba un acceso diferente, se pasan los valores como parámetros. Solo cambian los últimos dígitos del accessId y serial:

PowerShell — con parámetros personalizados
.\run_newman.ps1 `
    -AccessIdEntel "03-TESTPREPROD-DIR02873675-9" `
    -SerialEntel   "SCOM13032002" `
    -AccessIdCVTR  "02-TESTPREPROD-DIR02803674-002" `
    -SerialCVTR    "SCOM13022002"
ParámetroDescripciónEjemplo
-AccessIdEntelID de acceso para Entel. Cambiar los últimos dígitos en cada prueba.03-TESTPREPROD-DIR02873675-9
-SerialEntelSerial del equipo Entel.SCOM13032002
-SpeedEntelPlan de velocidad Entel. Opcional.940/940
-AccessIdCVTRID de acceso para ClaroVTR.02-TESTPREPROD-DIR02803674-002
-SerialCVTRSerial del equipo ClaroVTR.SCOM13022002
-SpeedCVTRPlan de velocidad ClaroVTR. Opcional.600/600

3.4 Qué pasa mientras ejecuta

El script lanza Entel y ClaroVTR al mismo tiempo (en paralelo). Durante la ejecución el terminal muestra el progreso. El proceso dura aproximadamente 60 a 90 segundos.

ℹ Es normal que el terminal quede "en silencio" un momento — está esperando que ambas ejecuciones terminen. No cerrar la ventana.

4Resultados — cómo ver los reportes

Al terminar, el script genera dos archivos HTML dentro de la carpeta Kommand:

Entel (VNO 03)
reporte_entel_extra.html
ClaroVTR (VNO 02)
reporte_clarovtr_extra.html

Abrir cualquiera de los dos archivos con doble clic — se abre en el navegador (Chrome, Edge, etc.).

Qué muestra el reporte

SecciónQué muestra
SummaryResumen total: cuántos tests pasaron, cuántos fallaron, tiempo total.
Failed TestsDetalle de cada test que falló con el mensaje de error exacto.
Request / ResponseEl body completo de cada request enviado y su respuesta.
Console LogLos logs internos con el resultado de cada operación: código de retorno, descripción, ID del acceso procesado.
⚠ Fallas esperadas: Los requests 01 (DevMod SYNC) y 04 (Modification ASYNC SSAA) siempre fallan en el entorno PRE-PROD actual. Esto es un problema conocido del servidor, no del script ni de los datos de prueba.
